**変更前**:
- フローパターンに応じて表示される納期が変わる

- ✅ フローパターンに関係なく全て表示

### 3. 納期タイプの追加・削除機能強化 ✅

**機能**:
- ✅ 「+ 納期タイプを追加」ボタン
- ✅ カスタム納期タイプを追加可能
- ✅ 追加した納期タイプは削除可能
- ✅ デフォルトの4種類は削除不可

**追加方法**:
1. 「+ 納期タイプを追加」ボタンをクリック
2. プロンプトで納期タイプ名を入力
   - 例：「通関から配送先への納期」
   - 例：「倉庫から港への納期」
3. 入力フィールドが追加される

**削除方法**:
1. カスタム納期タイプの右上に「削除」ボタンが表示される
2. クリックすると確認ダイアログが表示される
3. 確認すると削除される

**制限**:
- デフォルトの4種類の納期は削除できない
- カスタム納期のみ削除可能

**UI**:
```
┌─────────────────────────────────┐
│ 通関から配送先への納期    [削除] │  ← カスタム納期
│ [年/月/日 ▼]                   │
└─────────────────────────────────┘

┌─────────────────────────────────┐
│ メーカーから梱包業者への納期     │  ← デフォルト納期（削除不可）
│ [年/月/日 ▼]                   │
└─────────────────────────────────┘
```

  shipment_id INT NOT NULL,
  packing_company_id INT NOT NULL,
  PRIMARY KEY (shipment_id, packing_company_id),
  FOREIGN KEY (shipment_id) REFERENCES shipments(id) ON DELETE CASCADE,
  FOREIGN KEY (packing_company_id) REFERENCES companies(id) ON DELETE CASCADE
);

CREATE INDEX idx_shipment_packing_companies_shipment ON shipment_packing_companies(shipment_id);
