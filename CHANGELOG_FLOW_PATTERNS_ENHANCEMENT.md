# 業務フローパターンの追加・修正機能の強化

## 日時
2026年1月14日

## 実装内容

### 🎯 概要

業務フローパターンのマスタデータを一元管理し、出荷計画作成画面から直接フローパターンマスタ管理画面にアクセスできるようにしました。新しいパターンの追加や既存パターンの修正が簡単にできるようになりました。

---

### 2. 初期登録されているフローパターン

| ID | パターン名 | コード | 梱包業者 | 説明 |
|----|-----------|--------|---------|------|
| 1 | 直送（メーカー → フォワーダー） | direct | なし | メーカーから直接フォワーダーに配送 |
| 2 | 梱包業者経由（メーカー → 梱包業者 → フォワーダー） | via_packing | あり | メーカーから梱包業者を経由してフォワーダーに配送 |
| 3 | 複数拠点経由（メーカー → 梱包業者A → 梱包業者B → フォワーダー） | multi_packing | あり | 複数の梱包業者を経由して配送する特殊ルート |
| 4 | 国内配送のみ（メーカー → 梱包業者） | domestic_only | あり | 国内での梱包・保管のみ（輸出なし） |

## 🔄 データの流れ

### 1. マスタデータの一元管理

```
frontend/data/flow-patterns.ts
         ↓
         ├─→ 出荷計画作成画面（読み込み）
         │   └─ フローパターン選択ドロップダウン
         │
         └─→ フローパターンマスタ管理画面（読み込み・更新）
             └─ 追加・編集・削除機能
```

---

### 2. 出荷計画作成時の流れ

```
① 出荷計画作成画面を開く
   ↓
② フローパターンのドロップダウンで選択
   ↓
③ 「新しいパターンが必要」と判断
   ↓
④ 🔧 フローパターンマスタ管理 をクリック
   ↓
⑤ 新しいタブでマスタ管理画面が開く
   ↓
⑥ 「+ 新規パターンを追加」をクリック
   ↓
⑦ パターン情報を入力
   - パターン名
   - コード
   - 説明
   - 梱包業者の有無
   ↓
⑧ 「追加」をクリック
   ↓
⑨ 出荷計画作成画面に戻る
   ↓
⑩ ブラウザをリロード（F5）
   ↓
⑪ 新しいパターンが選択肢に表示される
```

## 💡 活用例

### シーン1: 新しいフローパターンを追加

**要件**: 
- メーカーから複数の梱包業者を経由して配送するパターンが必要

**手順**:

1. **出荷計画作成画面で「🔧 フローパターンマスタ管理」をクリック**

2. **マスタ管理画面で「+ 新規パターンを追加」をクリック**

3. **パターン情報を入力**:
   ```
   パターン名: 複数拠点経由（メーカー → 梱包業者A → 梱包業者B → フォワーダー）
   コード: multi_packing
   説明: 複数の梱包業者を経由して配送する特殊ルート
   ☑ 梱包業者を経由する
   ```

4. **「追加」をクリック**

5. **出荷計画作成画面に戻ってリロード（F5）**

6. **新しいパターンが選択肢に表示される** ✅

---

### シーン2: 既存パターンの説明を修正

**要件**: 
- 「直送」パターンの説明をより詳細にしたい

**手順**:

1. **フローパターンマスタ管理画面を開く**

2. **「直送（メーカー → フォワーダー）」の「編集」ボタンをクリック**

3. **説明を修正**:
   ```
   変更前: メーカーから直接フォワーダーに配送
   変更後: メーカーから直接フォワーダーに配送（梱包作業なし、最速ルート）
   ```

4. **「更新」をクリック**

5. **出荷計画作成画面で確認** ✅

---

### シーン3: 使わなくなったパターンを削除

**要件**: 
- テスト用に作成したパターンを削除

**手順**:

1. **フローパターンマスタ管理画面を開く**

2. **削除したいパターンの「削除」ボタンをクリック**

3. **確認ダイアログで「OK」をクリック**

4. **パターンが削除される** ✅

## 🎯 フローパターンの構造

### フロー図の表示パターン

#### パターン1: 直送（梱包業者なし）

```
┌─────────┐    ┌────────────┐    ┌────────┐
│メーカー │ → │フォワーダー│ → │配送先  │
└─────────┘    └────────────┘    └────────┘
```

**特徴**:
- 最速ルート
- コスト最小
- 梱包作業なし

---

#### パターン2: 梱包業者経由（梱包業者あり）

```
┌─────────┐    ┌──────────┐    ┌────────────┐    ┌────────┐
│メーカー │ → │梱包業者  │ → │フォワーダー│ → │配送先  │
└─────────┘    └──────────┘    └────────────┘    └────────┘
```

**特徴**:
- 商品のリパック可能
- ラベル貼り付け作業
- 品質チェック

---

#### パターン3: 複数拠点経由（複数の梱包業者）

```
┌─────────┐    ┌────────────┐    ┌────────────┐    ┌────────────┐    ┌────────┐
│メーカー │ → │梱包業者A   │ → │梱包業者B   │ → │フォワーダー│ → │配送先  │
└─────────┘    └────────────┘    └────────────┘    └────────────┘    └────────┘
```

**特徴**:
- 複雑な梱包作業
- 複数ステップの品質チェック
- 特殊なルート

---

#### パターン4: 国内配送のみ（輸出なし）

```
┌─────────┐    ┌──────────┐
│メーカー │ → │梱包業者  │
└─────────┘    └──────────┘
```

**特徴**:
- 国内保管のみ
- 輸出手続きなし
- 一時保管用
