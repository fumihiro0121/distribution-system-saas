# 開発計画書

## プロジェクト概要

### プロジェクト名
物流管理システムSaaS（Distribution Management System SaaS）

### プロジェクト期間
- **計画・設計フェーズ**: 2ヶ月
- **MVP開発フェーズ**: 4ヶ月
- **テスト・改善フェーズ**: 2ヶ月
- **本番リリース**: 8ヶ月後

### プロジェクト目標
メーカー、梱包業者、フォワーダー間の物流プロセスを一元管理し、トレーサビリティとオペレーション効率を向上させるSaaSプラットフォームの構築

### Phase 1: MVP開発（3-6ヶ月目）

#### 開発スコープ
MVPでは以下の機能を実装：

**必須機能**
1. ユーザー管理・認証
   - ログイン/ログアウト
   - パスワードリセット
   - ユーザー登録・編集
   - 役割ベースのアクセス制御

2. 出荷計画管理
   - 出荷計画の作成・編集・削除
   - 出荷計画一覧・詳細表示
   - ステータス管理

3. 商品マスタ（拡張版） ⭐
   - 基本情報（商品コード、商品名）
   - 識別コード（JANコード、HSコード）
   - 価格情報（原価、売価、通貨）
   - Amazon情報（ASIN、SKU、FNSKU）
   - サイズ・重量情報（容積重量自動計算）
   - セット商品管理（2個セット、3個セットなど）

4. 段ボール管理
   - 段ボール情報の登録・編集
   - 商品の割当
   - バーコード/QRコード生成
   - ステータス管理

5. パレット管理
   - パレット情報の登録・編集
   - 段ボールの割当
   - 積付図表示（簡易版）
   - ステータス管理

6. 基本的な作業指示機能
   - 作業指示の表示
   - 作業報告（テキスト + 写真）
   - ステータス更新

7. 基本的な進捗管理
   - ダッシュボード（基本統計）
   - 出荷進捗表示
   - 簡易トラッキング

8. マスタ管理
   - 商品マスタ（詳細情報含む）
   - 取引先マスタ
   - サイズマスタ
   - 配送先規定マスタ（Amazon FBA、一般輸出） ⭐

**スプリント計画（2週間スプリント × 8回）**

## 開発標準・ベストプラクティス

### コーディング規約
1. **TypeScript厳格モード使用**
   ```json
   {
     "strict": true,
     "noImplicitAny": true,
     "strictNullChecks": true
   }
   ```

2. **命名規則**
   - コンポーネント: PascalCase (`UserProfile.tsx`)
   - 関数・変数: camelCase (`getUserData`)
   - 定数: UPPER_SNAKE_CASE (`MAX_RETRY_COUNT`)
   - ファイル名: kebab-case (`user-profile.tsx`)

3. **ディレクトリ構造**
   ```
   src/
   ├── components/       # 共通コンポーネント
   ├── features/         # 機能別コンポーネント
   ├── pages/            # ページコンポーネント
   ├── hooks/            # カスタムフック
   ├── utils/            # ユーティリティ関数
   ├── types/            # 型定義
   ├── services/         # API呼び出し
   └── styles/           # スタイル
   ```

### Git戦略
- **ブランチ戦略**: Git Flow
  - `main`: 本番環境
  - `develop`: 開発環境
  - `feature/*`: 機能開発
  - `release/*`: リリース準備
  - `hotfix/*`: 緊急修正

- **コミットメッセージ**: Conventional Commits
  ```
  feat: 段ボール管理機能を追加
  fix: パレット割当時のバグを修正
  docs: API仕様書を更新
  test: 出荷計画のテストを追加
  refactor: 認証ロジックをリファクタリング
  ```

### コードレビュー
- 全てのコードはプルリクエスト経由
- 最低1名のレビュー承認が必要
- 自動テストがパスしていることを確認
- レビューは24時間以内に実施

### テスト戦略
- **単体テスト**: 関数・コンポーネント単位
- **統合テスト**: API・データベース統合
- **E2Eテスト**: 主要ユーザーフロー
- **目標カバレッジ**: 80%以上
