# 段ボールサイズ推奨システム

## 概要

商品の出荷実績データを基に、最適な段ボールサイズを自動推奨するシステムです。

## 機能

1. **Googleスプレッドシートからデータ取得**
   - 商品情報、実績段ボールサイズ、梱包数などを取得

2. **段ボールサイズ推奨**
   - 商品の数量パターンごとに最適な段ボールを提案
   - 実績データを基に、1袋、5袋、10袋、20袋などの様々な数量で推奨

3. **マスタデータSQL生成**
   - 商品マスタのINSERT文を生成
   - 段ボールサイズマスタのINSERT文を生成
   - 商品と段ボールの推奨マッピングを生成

## 段ボール推奨ロジック

### 基本的な考え方

1. **実績データの活用**
   - スプレッドシートの「段ボール縦センチ」「段ボール横センチ」「段ボール高さセンチ」は、
     その商品を実際にそのサイズの段ボールで輸出した実績を示す
   - 例: 「黒ゴマアーモンドきな粉150g×10袋セット」が「34.1×22.6×10.9cm」の段ボールで
     輸出された実績がある場合、1袋セットを輸出する際もこのサイズを選択肢として提案

2. **数量パターンの生成**
   - 1袋セット商品: 1袋、5袋、10袋、20袋、30袋、50袋、実績梱包数
   - ○袋セット商品: 1セット、2セット、5セット、10セットなど

3. **段ボールサイズの計算**
   - 実績データから1セットあたりの必要スペースを計算
   - 必要なセット数に応じて、10%の余裕を持たせたサイズを算出
   - ダンボールワンのデータベースから最適なサイズを検索

4. **優先順位**
   - 実績のある段ボールサイズを優先
   - 容積が小さいものから順に提案（コスト削減）
   - 上位3つの候補を提示

### 例: 黒ゴマアーモンドきな粉150g×1袋セット

実績データ:
- 110袋を53.0×38.0×35.0cmの段ボールに梱包

推奨:
- 1袋の場合: 664×184×148mm の段ボール
- 10袋の場合: 664×184×148mm の段ボール
- 50袋の場合: 634×194×268mm の段ボール
- 110袋の場合: 実績段ボール（53.0×38.0×35.0cm）を含む候補

## データベーススキーマ

### product_master（商品マスタ）

- `id`: 主キー
- `product_name`: 商品名
- `sku`: SKU
- `jan_code`: JANコード
- `asin`: ASIN
- `fnsku`: FNSKU
- `hs_code`: HSコード
- `supplier`: 仕入先
- `unit_weight`: 単位重量（kg）
- `unit_weight_lb`: 単位重量（ポンド）
- `standard_box_quantity`: 標準箱入数
- `category`: カテゴリ

### carton_size_master（段ボールサイズマスタ）

- `id`: 主キー
- `carton_code`: 段ボール商品番号（例: MA120-398）
- `carton_name`: 段ボール商品名
- `supplier`: 仕入先（例: ダンボールワン）
- `inner_length_mm`: 内寸長さ（mm）
- `inner_width_mm`: 内寸幅（mm）
- `inner_height_mm`: 内寸高さ（mm）
- `outer_total_cm`: 外形三辺合計（cm）
- `thickness`: 厚み
- `material_type`: 形式
- `delivery_size`: 宅配サイズ
- `product_url`: 商品URL
- `unit_price`: 単価
- `currency`: 通貨

### product_carton_recommendation（商品段ボール推奨マッピング）

- `product_id`: 商品ID（外部キー）
- `carton_id`: 段ボールID（外部キー）
- `quantity_from`: 数量範囲（開始）
- `quantity_to`: 数量範囲（終了）
- `priority`: 優先順位（1が最優先）
- `is_historical`: 実績があるかどうか

## 注意事項

1. **スプレッドシートの公開設定**
   - Googleスプレッドシートは「リンクを知っている全員が閲覧可能」に設定する必要があります

2. **段ボールデータの更新**
   - ダンボールワンのウェブサイト構造が変更された場合、スクレイパーの修正が必要になる可能性があります

3. **推奨ロジックの調整**
   - 余裕率（現在10%）やその他のパラメータは、実際の梱包作業の経験に基づいて調整してください

4. **データベースへのインポート**
   - `master_data_insert.sql`をデータベースにインポートする前に、既存データのバックアップを取ってください

## トラブルシューティング

### エラー: "Request failed with status code 401"

- スプレッドシートが非公開になっています
- 共有設定を確認してください

### エラー: "No products found on page 1"

- ウェブサイトの構造が変更された可能性があります
- `cardboard_scraper_final.js`のセレクタを確認してください

### 推奨段ボールが見つからない

- 実績データの段ボールサイズが大きすぎる可能性があります
- `recommend_carton_sizes.js`の余裕率を調整してください

## インタラクティブUI（フロントエンド）

### CartonSelector コンポーネント

`frontend/app/components/CartonSelector.tsx`

段ボールを選択・調整するためのReactコンポーネントです。

#### 主な機能

1. **段ボール候補の表示**
   - 実績データに基づく優先順位付き候補リスト
   - 各段ボールの収容数、サイズ、宅配サイズを表示
   - 実績ありバッジ、優先度バッジ

2. **複数種類の段ボール選択**
   - 候補リストから段ボールを追加
   - 同じ段ボールを複数回追加可能
   - 各段ボールの箱数を個別に調整

3. **自動提案機能**
   - **第一候補で満たす**: 最優先の段ボールのみで目標数量を満たす
   - **最適化された組み合わせ**: 過剰を最小化する組み合わせを自動計算

4. **リアルタイム集計**
   - 選択中の段ボールの合計箱数・合計袋数を表示
   - 残り数量をリアルタイムで表示
   - 過剰/不足を色分けで表示

#### 使用例

```typescript
import CartonSelector from '@/app/components/CartonSelector';

<CartonSelector
  productName="黒ゴマアーモンドきな粉150g×1袋セット"
  targetQuantity={1000}
  cartonOptions={cartonOptions}
  onSelectionChange={(selectedCartons, remaining) => {
    console.log('選択された段ボール:', selectedCartons);
    console.log('残り数量:', remaining);
  }}
/>
```

## 改良版推奨ロジック（advanced）

### recommend_carton_advanced.js

`scraping/recommend_carton_advanced.js`

同じベース商品名の全ての実績データを活用する改良版です。

#### 主な改善点

1. **全実績データの活用**
   - 「×1袋セット」「×2袋セット」「×10袋セット」など、同じ商品の全実績を使用
   - 各実績から最適な段ボールを提案

2. **1袋セット換算**
   - 「×10袋セット」の実績を「×1袋セット」に換算
   - 例: 341×226×109mmに10袋入った実績 → 1袋セット換算で10袋入り

3. **柔軟な容積計算**
   - 実績データから1袋あたりの必要容積を計算
   - 他の段ボールサイズでの収容数を推定

4. **優先順位付け**
   - 実績段ボールを優先度1
   - 実績に近いサイズの段ボールを優先度2以降

#### 実行方法

```bash
# デフォルト（1000袋）
node recommend_carton_advanced.js

# カスタム数量
node recommend_carton_advanced.js 500
```

## ユースケース例

### ケース1: 黒ゴマアーモンドきな粉150g×1袋セット 1000袋

#### 要件
- 実績段ボール（341×226×109mm、10袋入り）を第一候補とする
- 複数種類の段ボールを組み合わせ可能
- 箱数を調整して過剰を最小化

#### 推奨される組み合わせ

**パターン1: 実績段ボールのみ**
- 341×226×109mm × 100箱 = 1000袋
- 過剰: 0袋

**パターン2: 実績段ボール + 大型段ボール**
- 341×226×109mm × 60箱 = 600袋
- 590×470×230mm × 4箱 = 456袋
- 合計: 1056袋（過剰: 56袋）

**パターン3: カスタム調整**
- ユーザーが箱数を調整して最適化
- 例: 341×226×109mm × 70箱 = 700袋
      590×470×230mm × 3箱 = 342袋
      残り: -42袋（42袋不足） → さらに調整

### ケース2: きな粉150g×1袋セット 1000袋

#### 推奨される組み合わせ

**パターン1: 最適化された組み合わせ（過剰0袋）**
- 584×224×148mm × 14箱 = 490袋
- 624×204×148mm × 15箱 = 510袋
- 合計: 29箱、1000袋（過剰: 0袋）
